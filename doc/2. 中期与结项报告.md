## 一、项目概述 

### 1. 项目背景
  随着⼈们对健康饮⻝的重视，绿⾊有机⻝品需求⽇益增⻓。然⽽传统线下购买存在信息不透明、渠道有限、配
送效率低等问题。开发⼀个基于WEB的绿⾊有机⻝品配送系统，可以利⽤⽹络实现商品展⽰、在线下单和基础
管理，解决信息零散、购买不便的问题，提升⽤户体验和管理效率。
### 2. 系统目标
- 实现普通用户注册、登录、商品浏览搜索、个人信息管理等基础功能，后续扩展购物车、下单、订单查看等完整购物流程；
- 实现管理员对商品信息的增删改查、上下架管理，后续扩展订单管理、用户账号管理等能力；
- 实现配送人员订单查询、配送状态更新的基础能力；
- 保障数据安全与接口权限隔离，满足基础性能要求，在四周内完成可演示、可运行的原型系统。
### 3. 开发环境

|开发工具	|IntelliJ IDEA、VS Code、Git、Postman|
|后端运行环境|JDK 17、Spring Boot 3.5.14、内置 Tomcat|

## 二、需求分析 

### 1. 功能需求
- 核心角色与业务边界
- 本系统分为三类核心角色，中期阶段已完成各角色的基础权限与核心功能边界定义，具体如下：


|角色	|中期已完成功能	|
|---------|----------|
|普通用户 |	账号注册、登录退出、个人信息查看与修改、密码修改、商品列表浏览与搜索|	
|管理员	|账号登录、商品信息管理（增删改查、上下架）、个人信息管理|
|配送人员|	账号登录、个人信息管理、菜单权限隔离|


- 核心业务流程
- 账号全流程：用户注册 → 账号登录 → 身份鉴权 → 页面权限分配 → 退出登录
- 个人信息管理：登录后查看个人信息 → 修改基础资料 / 登录密码 → 数据同步更新
- 商品管理流程：管理员登录 → 商品列表查询 / 筛选 / 搜索 → 商品新增 / 编辑 / 删除 / 上下架操作
### 2. 非功能需求
- 性能要求：页面加载响应时间≤2 秒，接口请求响应时间≤3 秒，登录操作响应时间≤1 秒；
- 安全要求：用户身份通过 JWT 令牌鉴权，接口全局权限拦截，不同角色菜单与接口权限隔离，密码传输与存储做安全处理，未登录用户禁止访问系统内部页面与接口；
- 兼容性要求：适配 Chrome、Edge 最新版本浏览器，PC 端主流分辨率下页面布局正常展示，无样式错乱与功能异常。

## 三、系统设计  

### 1. 系统架构
- 本系统采用前后端分离的分层架构，整体分为三层，各层职责清晰、解耦设计：
- 前端展示层：基于 Vue2 构建单页面应用，通过 Element UI 实现页面组件化开发，Vue Router 实现路由管理与权限控制，Axios 完成前后端数据通信；
- 后端服务层：基于 Spring Boot 搭建后端服务，采用经典 MVC 分层架构，分为 Controller 接口层、Service 业务层、Mapper 数据访问层，通过拦截器实现全局身份鉴权，JWT 完成无状态身份认证；
- 数据持久层：基于 MySQL 8.0 实现业务数据存储，MyBatis 完成数据库操作与对象映射，通过约束与索引保障数据完整性与查询效率。

## 四、系统实现  

### 1. 关键技术

#### 1.1 前端核心技术实现
- **核心框架**：Vue 2.6.14 + Element UI 2.15.14 + Vue Router 3.6.5 + Axios 1.15.1 + vue-cli 5.0.0
- **核心逻辑**：
  - 自定义表单校验规则，实现用户名、手机号、密码等字段的实时验证
  - 基于角色的菜单权限控制，不同角色显示不同导航菜单
  - 分页查询联动，支持页码切换和每页数量调整
  - 全局请求拦截器，自动注入JWT令牌到请求头
  - 路由守卫拦截，未登录用户重定向到登录页
  - 响应拦截器统一处理错误码，提供友好的错误提示
- **已解决难点**：
  - 前后端跨域代理配置，通过vue.config.js配置devServer代理
  - 多角色菜单隔离，根据用户role动态渲染菜单项
  - 表单校验状态重置，弹窗关闭时自动清空校验状态
  - 全局异常统一处理，捕获网络错误和业务错误

#### 1.2 后端核心技术实现
- **核心框架**：Spring Boot 3.5.14 + MyBatis 3.0.5 + JJWT 0.12.5 + Maven
- **核心逻辑**：
  - JWT令牌生成与校验，使用JJWT库实现无状态身份认证
  - 全局接口权限拦截，通过HandlerInterceptor实现Token验证
  - 统一响应结果封装，使用Result类统一返回格式
  - 密码安全校验，注册和登录时验证用户名和手机号唯一性
  - 驼峰命名自动映射，MyBatis配置mapUnderscoreToCamelCase
  - 事务管理，关键业务操作使用@Transactional保证数据一致性
- **已解决难点**：
  - 无状态身份认证，通过JWT实现分布式会话管理
  - 全局接口统一鉴权，白名单机制放行登录注册等公开接口
  - 前后端接口格式统一，定义统一的Result响应结构
  - 用户账号唯一性保障，注册和修改时检查用户名和手机号重复
  - 余额支付原子性操作，使用数据库事务保证扣款和订单状态更新的原子性
  - 配送员自动分配，支付成功后随机分配空闲配送员并更新状态

#### 1.3 数据库核心技术实现
- **核心技术**：MySQL 8.0（InnoDB引擎）+ B+树索引 + MyBatis驼峰下划线自动映射
- **核心设计**：
  - 唯一性约束，username和phone字段设置唯一索引
  - 合理字段类型适配，金额使用DECIMAL保证精度
  - 非空约束，关键字段设置NOT NULL
  - 主键自增，所有表使用AUTO_INCREMENT主键
  - 分页查询优化，使用LIMIT OFFSET实现高效分页
  - 外键关联，订单、购物车等表通过外键关联用户和商品
- **已解决难点**：
  - 高频查询字段性能优化，为常用查询字段添加索引
  - 字段名与属性名自动映射，MyBatis配置自动转换

#### 3.1 JWT令牌生成与验证（后端）
java // JwtUtils.java - JWT工具类 public class JwtUtils { private static final String SECRET_KEY = "your-secret-key"; private static final long EXPIRATION_TIME = 24 * 60 * 60 * 1000; // 24小时
// 生成JWT令牌
public static String generateToken(Map<String, Object> claims) {
    return Jwts.builder()
            .claims(claims)
            .expiration(new Date(System.currentTimeMillis() + EXPIRATION_TIME))
            .signWith(Keys.hmacShaKeyFor(SECRET_KEY.getBytes()))
            .compact();
}

// 解析JWT令牌
public static Claims parseToken(String token) {
    return Jwts.parser()
            .verifyWith(Keys.hmacShaKeyFor(SECRET_KEY.getBytes()))
            .build()
            .parseSignedClaims(token)
            .getPayload();
}
}

java // OrderServiceImpl.java - 订单服务实现 @Override @Transactional public Result createOrder(HttpServletRequest request, OrderCreateRequest orderRequest) { // 1. 获取用户信息 String token = request.getHeader("token"); Claims claims = JwtUtils.parseToken(token); Integer userId = (Integer) claims.get("id");
// 2. 验证购物车商品
List<Cart> carts = cartMapper.selectByUserId(userId);
if (carts == null || carts.isEmpty()) {
    return Result.error(400, "购物车为空");
}

// 3. 创建订单
Order order = new Order();
order.setUserId(userId);
order.setReceiver(orderRequest.getReceiver());
order.setPhone(orderRequest.getPhone());
order.setAddress(orderRequest.getAddress());
order.setPaymentType(orderRequest.getPaymentType());
order.setRemark(orderRequest.getRemark());

// 4. 计算订单金额
BigDecimal goodsTotal = BigDecimal.ZERO;
List<OrderItemEntity> orderItems = new ArrayList<>();

for (Cart cart : carts) {
    Product product = productMapper.selectById(cart.getProductId());
    if (product == null || product.getStatus() == 0) {
        return Result.error(400, "商品不存在或已下架");
    }
    
    // 检查库存
    if (product.getStock() < cart.getQuantity()) {
        return Result.error(400, "商品库存不足");
    }
    
    // 扣减库存
    productMapper.deductStock(product.getId(), cart.getQuantity());
    
    // 计算小计
    BigDecimal subtotal = product.getPrice().multiply(new BigDecimal(cart.getQuantity()));
    goodsTotal = goodsTotal.add(subtotal);
    
    // 创建订单项
    OrderItemEntity item = new OrderItemEntity();
    item.setProductId(product.getId());
    item.setProductName(product.getName());
    item.setPrice(product.getPrice());
    item.setQuantity(cart.getQuantity());
    item.setSpecification(product.getSpecification());
    item.setImageUrl(product.getImageUrl());
    item.setSubtotal(subtotal);
    orderItems.add(item);
}

// 5. 计算运费和总额
BigDecimal deliveryFee = goodsTotal.compareTo(new BigDecimal("50")) >= 0 ? BigDecimal.ZERO : new BigDecimal("8");
BigDecimal totalAmount = goodsTotal.add(deliveryFee);

order.setGoodsTotal(goodsTotal);
order.setDeliveryFee(deliveryFee);
order.setTotalAmount(totalAmount);
order.setStatus(0); // 待支付

// 6. 保存订单
orderMapper.insertOrder(order);

// 7. 保存订单项
for (OrderItemEntity item : orderItems) {
    item.setOrderId(order.getId());
    orderMapper.insertOrderItem(item);
}

// 8. 清空购物车
cartMapper.deleteByUserId(userId);

return Result.success(200, "订单创建成功", order.getId());
}

3. 可填写订单备注
4. 点击"提交订单"完成下单
5. 如选择余额支付且余额充足，自动扣款并分配配送员
6. 如余额不足，系统提示充值

**查看订单**
1. 点击导航栏"我的订单"
2. 可按状态筛选订单
3. 待支付订单可执行"立即支付"或"取消订单"
4. 配送中订单可查看配送进度
5. 待收货订单可"确认收货"
6. 已完成订单可"评价"商品

**个人中心**
1. 点击导航栏"个人中心"
2. 查看个人信息和账户余额
3. 点击"立刻充值"进行账户充值
4. 点击"修改资料"修改用户名和手机号
5. 点击"修改密码"修改登录密码

6. 点击"上架/下架"改变商品状态

**订单管理**

2. 可按用户姓名、配送员姓名搜索

4. 查看所有订单详细信息

**用户管理**

**配送员管理**
