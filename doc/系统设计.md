# 系统设计
## 一、系统架构
本系统采用前后端分离的分层架构，整体分为三层，各层职责清晰、解耦设计：

### 1. 前端展示层
基于 Vue2 构建单页面应用，通过 Element UI 实现页面组件化开发，Vue Router 实现路由管理与权限控制，Axios 完成前后端数据通信。

### 2. 后端服务层
基于 Spring Boot 搭建后端服务，采用经典 MVC 分层架构，分为 Controller 接口层、Service 业务层、Mapper 数据访问层，通过拦截器实现全局身份鉴权，JWT 完成无状态身份认证。

### 3. 数据持久层
基于 MySQL 8.0 实现业务数据存储，MyBatis 完成数据库操作与对象映射，通过约束与索引保障数据完整性与查询效率。

#### 数据流向说明
用户操作 → 前端页面组件 → Axios网络请求（携带JWT令牌）→ 后端Token拦截器校验 → Controller接口接收请求 → Service处理业务逻辑 → Mapper访问数据库 → 数据库返回数据 → Mapper → Service → Controller → Axios响应拦截器处理 → 前端页面组件渲染 → 展示给用户

## 二、模块设计
|模块名称|核心职责|
|------|-------|
|认证授权模块|处理三类角色的注册、登录、退出逻辑，JWT 令牌生成与解析，全局接口权限拦截|
|用户信息模块|处理用户基础信息查询、资料修改、密码更新等核心逻辑|
|商品管理模块|处理商品信息的增删改查、上下架状态管理、关键词与分类筛选、分页查询|
|前端路由模块|实现页面路由跳转、路由守卫、未登录拦截、角色菜单权限隔离|
|全局工具模块|统一接口请求封装、统一响应格式、全局异常处理、表单校验规则封装|

## 三、数据库设计
[E-R图](./asset/er.jpg)


- 主要数据表设计
  **1. user 用户表**
  系统核心基础表，承载三类角色的账号信息。

|字段名|类型|说明|约束|
|------|----|----|----|
|id|INT|用户ID|主键，自增|
|username|VARCHAR(50)|用户名|唯一，非空|
|password|VARCHAR(100)|密码|非空|
|phone|VARCHAR(11)|手机号|唯一，非空|
|id_number|VARCHAR(18)|身份证号|可空|
|role|VARCHAR(20)|角色标识（user/admin/PEISONG）|非空，默认'user'|
|status|ENUM('配送中','空闲')|配送员状态|可空，仅PEISONG有效|
|money|DECIMAL(10,2)|账户余额|默认0.00|

**2. products 商品表**
存储商品信息，支持上下架管理。

|字段名|类型|说明|约束|
|------|----|----|----|
|id|INT|商品ID|主键，自增|
|name|VARCHAR(100)|商品名称|非空|
|category|VARCHAR(50)|商品分类|非空|
|price|DECIMAL(10,2)|商品价格|非空|
|stock|INT|库存数量|非空，默认0|
|image_url|VARCHAR(255)|商品图片URL|可空|
|description|TEXT|商品描述|可空|
|status|TINYINT|上架状态（1上架/0下架）|非空，默认1|
|specification|VARCHAR(50)|规格|可空|
|click_count|INT|点击量|默认0|

**3. cart 购物车表**
存储用户购物车中的商品信息。

|字段名|类型|说明|约束|
|------|----|----|----|
|id|INT|购物车ID|主键，自增|
|user_id|INT|用户ID|外键，非空|
|product_id|INT|商品ID|外键，非空|
|quantity|INT|商品数量|非空，默认1|

**4. orders 订单表**
存储订单基本信息。

|字段名|类型|说明|约束|
|------|----|----|----|
|id|INT|订单ID|主键，自增|
|order_no|VARCHAR(50)|订单编号|唯一，非空|
|user_id|INT|用户ID|外键，非空|

|delivery_type|VARCHAR(20)|配送方式|非空|
|payment_type|VARCHAR(20)|支付方式（balance/cod）|非空|
|remark|VARCHAR(255)|订单备注|可空|
|goods_total|DECIMAL(10,2)|商品总额|非空|
|delivery_fee|DECIMAL(10,2)|运费|非空，默认0|
|total_amount|DECIMAL(10,2)|订单总额|非空|
|status|TINYINT|订单状态（0待支付/1配送中/2已送达/3已完成/4已取消）|非空，默认0|

存储用户对订单的评价信息。

|字段名|类型|说明|约束|
|------|----|----|----|
